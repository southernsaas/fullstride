# Contributing

Keep changes small and reviewable.

1. Create a branch from `main`.
2. Update the static HTML, CSS, JavaScript, or assets.
3. Run the site locally and test onboarding, lesson sequencing, quizzes,
   certificate issuance, refresh persistence, and mobile layout.
4. Open a pull request with a short description and screenshots for visual
   changes.

Do not commit learner data, customer data, credentials, private video URLs, or
unapproved Fullstride brand assets.
